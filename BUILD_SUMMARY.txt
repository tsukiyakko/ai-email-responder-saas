IMPLEMENTATION DETAILS
================================================================================

EMAIL GENERATION (/api/generate):
  • Accepts: customerEmail, companyName, senderName, notes, mode
  • Validates subscription status (trial/paid/expired)
  • Applies rate limiting (5/10min per user)
  • Calls Gemini API in parallel for 3 languages
  • Increments usage counters in Redis
  • Returns HTML 403 if limits exceeded
  • Returns 429 if rate limited
  • Returns generated replies in all 3 languages

USAGE TRACKING:
  Trial Users:
    - Redis key: trial:{userId}:total
    - Max 2 total generations for 1 day
    - Expiry tracked in Redis: trial_start:{userId}

  Paid Users:
    - Redis key: usage:{userId}:{YYYY-MM-DD}
    - Max 30 generations per day
    - Auto-expires after 24 hours

SUBSCRIPTION MANAGEMENT:
  Trial Initialization:
    - Auto-triggered on first /dashboard visit
    - Stores start date in Redis
    - Sets subscription status to "trial"

  Payment Flow:
    - User clicks upgrade → /api/stripe/checkout
    - Creates Stripe checkout session with user metadata
    - Redirects to Stripe-hosted checkout
    - After payment → Stripe webhook fires
    - Webhook updates Redis: subscription:{userId} = "active"

  Cancellation:
    - Stripe webhook: customer.subscription.deleted
    - Updates Redis: subscription:{userId} = "none"
    - User reverts to no subscription

RATE LIMITING:
  • Uses Upstash Ratelimit library
  • Sliding window: 5 requests per 10 minutes
  • Per-user basis (keyed by userId)
  • Returns HTTP 429 if exceeded
  • Applied only to /api/generate (not read operations)
